Design System Guide

The visual language for Family Fork. Colours are drawn from the natural palette of food — the warmth of spices, the brightness of citrus, the freshness of herbs. All tokens are defined as CSS variables for consistent implementation.

📋 Usage Rules

Guardrails to keep the system consistent as the product grows.

✅ Do
  • Use orange for all primary CTAs and active states
  • Use red only for spice indicators and destructive actions
  • Use yellow for passive info, previews, and annotations
  • Use green for success states, diet tags, and step circles
  • Use warm-tinted shadows rgba(60,20,0,...) always
  • Keep backgrounds warm: --bg (#fdf6ec) on all pages
  • Use --border (#f0d9b8) for all dividers
  • Reference tokens (var(--orange)), never raw hex values
🚫 Don't
  • Don't use cool greys or blue-tinted neutrals
  • Don't mix tag colour categories (e.g. yellow cuisine tag)
  • Don't use red as a primary button colour
  • Don't use yellow as standalone CTA — illegible contrast
  • Don't use gradients on body text or small labels
  • Don't add colours outside the four-hue palette without approval
  • Don't use hard black (#000) — use --text instead
  • Don't use pure blue shadows — keep them warm-tinted
